AI-Driven Image Cropping — How It Works

Junior analyzes guide images automatically — no manual annotation needed

1
Guide Uploaded or Fetched
PDF pages rendered as images. Photos kept as-is. All images stored in Vercel Blob with their page/position metadata.
2
Claude Vision Analyses Each Image
For each step, Claude receives: the step text + the associated image(s). It returns a bounding box — the region most relevant to that step.
"For step 3 (knead the dough), highlight: { x: 120, y: 45, w: 210, h: 160 }"
3
Bounding Boxes Saved with Each Step
Each guide step stores: text · associated image URL · AI crop coordinates. This happens once at guide creation — not at session time.
4
During Session — Step Changes
When user advances to a step that has an image, the UI renders the image with a highlight overlay over the AI-selected crop region. Junior can also say "take a look at the highlighted part."
[ guide image ]
AI crop overlay
User can still pinch/zoom freely — the highlight is a guide, not a lock
What this adds to the stack:  Vercel Blob (image storage) · PDF-to-image rendering (pdf.js or Puppeteer) · one extra Claude vision call per step at guide creation time
