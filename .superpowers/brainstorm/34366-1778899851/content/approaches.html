Three Ways to Build Junior

Pick the approach that fits your 24-hour window and portfolio goals

A

Lean Web App — Next.js + Claude API

Stack: Next.js · Claude claude-sonnet-4-6 (vision) · Web Speech API · Vercel

One API key does everything: reads your uploaded PDF, extracts recipe steps, answers questions, and checks your camera photo. Web Speech API handles voice in-browser — no extra service needed.

Pros

  • Single API provider — simplest setup
  • Claude vision handles OCR + progress checking natively
  • Vercel deploy in under 5 minutes
  • Strong portfolio story: "AI-native from day one"

Cons

  • Web Speech API quality varies by browser
  • No persistent sessions across devices

⭐ Recommended for 24-hour build

B

Mobile-First PWA — React + Whisper + GPT-4o

Stack: React PWA · OpenAI Whisper (voice) · GPT-4o (vision) · Vercel

Progressive Web App optimized for mobile. Whisper gives high-accuracy voice recognition; GPT-4o handles vision. Feels native on your phone.

Pros

  • Best voice accuracy (Whisper)
  • Native-feel mobile camera UX
  • GPT-4o is strong for multimodal tasks

Cons

  • Two API providers adds complexity
  • Whisper adds round-trip latency
  • More setup overhead in a 24-hour window
C

Rapid Prototype — Streamlit + Claude API

Stack: Python · Streamlit · Claude claude-sonnet-4-6 · Hugging Face Spaces

Python-first approach using Streamlit for UI. Fast to scaffold, easy to iterate. Better for data scientists than a polished consumer app.

Pros

  • Fastest to get running
  • Easy state management in Python
  • Free Hugging Face deployment

Cons

  • Voice is second-class in Streamlit
  • Desktop-only feel — doesn't match the hands-free use case
  • Less polished as a portfolio piece
