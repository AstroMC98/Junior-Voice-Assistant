Image Display — Two Design Directions

How does the user see and annotate guide images during a session?

A

Creator Annotates, User Views

The guide creator (you) crops and highlights regions when building the guide. During sessions, the annotated image is shown automatically at the right step — the user just sees what you decided to highlight.

Guide Editor — Step 3 "Knead the dough"
[ recipe photo ]
✏ Drag to crop · Click to annotate
During session:
Step card + highlighted image shows automatically. User hears Junior say "it should look like this — see the highlighted area."

Pros

  • Curated experience — creator controls the clarity
  • Simpler session UI (no user interaction needed)

Cons

  • More work for guide creator
B

User Crops On Demand

All guide images are available during a session. When Junior references a diagram or photo, it appears. The user can tap and drag to crop/zoom into the part they need — no pre-annotation required.

Session UI — Step 3 "Knead the dough"
[ recipe photo ]
📌 Pinch to zoom · drag to crop
During session:
Image pops up when relevant. User can zoom into any part. No setup needed by creator.

Pros

  • Zero friction for guide creators
  • User has full control mid-session

Cons

  • Harder to surface the right image at the right moment
C

Both — Creator Annotates + User Can Adjust

Creator pre-annotates the highlight, but the user can still pinch/zoom or drag the crop region during the session if they need a different angle. Best of both worlds.

Pros

  • Guided but flexible
  • Works for both detailed manuals and quick recipes

Cons

  • Most to build in 24 hours

⭐ Best long-term, but scope risk for 24h build
