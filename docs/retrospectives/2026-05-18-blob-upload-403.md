# Blob Upload 403 Retrospective

## Context

While creating a guide with image uploads, Blob storage returned:

```json
{
  "detail": "Blob upload failed 403: {\"error\":{\"code\":\"forbidden\",\"message\":\"Cannot get store id from token or header\"}}"
}
```

## What Happened

The Python backend loads `.env.local` during local development. `BLOB_READ_WRITE_TOKEN` was present, but its value in `.env.local` was wrapped in quotes. Our local env loader preserved those quotes instead of stripping them, so the Blob API received an invalid bearer token.

That meant the request sent `"vercel_blob_rw_..."` instead of `vercel_blob_rw_...`, and Vercel Blob could not extract the store ID from the token.

## Root Cause

The issue was not the Blob upload request structure itself. The real failure was a mismatch between:

- How `.env.local` values were written
- How the Python backend parsed those values

The local loader used raw string values from `.env.local` and did not normalize quoted secrets before placing them in `os.environ`.

## Fix

We introduced a small env utility that:

- Loads `.env.local` for local development
- Strips matching single or double quotes from env values
- Preserves unquoted values as-is

The FastAPI entry point now uses this shared loader instead of manually parsing `.env.local`.

## Prevention

- Normalize env values when loading local config files
- Add regression tests for quoted and unquoted env values
- Treat auth errors from third-party services as possible config parsing issues, not only credential issues
- Be careful when copying secrets from dashboards or generated files that may include quotes

## Learning

A valid-looking secret can still fail if it is parsed incorrectly before it reaches the SDK or API. When a service says it cannot identify a token or store, inspect the exact runtime shape of the value being sent, including invisible formatting like wrapping quotes or whitespace.

## Follow-Up

- Keep the regression test for env parsing in place
- Reuse the shared env loader for any future Python entry points
- Restart the local Python server after `.env.local` parsing changes so the updated values are reloaded
